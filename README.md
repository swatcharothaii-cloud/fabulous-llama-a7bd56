# ⏰ ระบบลงเวลาทำงาน + คำนวณ OT อัตโนมัติ (Attendance & OT Mini App)

มินิแอปเลียนแบบสไตล์ "Hello Boss" สำหรับบริษัทขนาดเล็ก-กลาง (20-100 คน) รองรับ:

- เช็คอิน/เช็คเอาท์ได้หลายครั้งต่อวัน (เช่น พักเที่ยงออกไปข้างนอกแล้วกลับมาทำงานต่อ)
- กะการทำงาน 3-4 ช่วงเวลา (เช้า/บ่าย/ดึก/ออฟฟิศ) กำหนดเองได้ทั้งหมด รวมกะที่ข้ามเที่ยงคืน
- คำนวณชั่วโมงทำงาน + OT อัตโนมัติ ตามหลักมาตรฐานแรงงานไทย (ปรับอัตราเองได้)
- ระบบลา (ลาป่วย/ลากิจ/ลาพักร้อน ฯลฯ) พร้อมอนุมัติ/ไม่อนุมัติจากแอดมิน
- สลับวันหยุด — ระบบจัดการวันทำงาน/วันหยุดที่สลับให้อัตโนมัติในการคำนวณ OT และรายงาน
- ดึงวันหยุดประจำปีของบริษัทมาใช้ในระบบ (แก้ไข/เพิ่มเองได้ที่หน้าแอดมิน)
- สรุปประจำวันให้พนักงานเห็นชั่วโมงทำงาน/OT ของตัวเองทันที
- รายงาน + ส่งออก Excel สำหรับฝ่ายบุคคล/บัญชี

รองรับการใช้งานเป็น **เว็บแอปทั่วไป** และ **LINE Mini App (LIFF)**

> ⚠️ **สำคัญมาก:** ตัวเลข/สูตรคำนวณ OT ที่มากับแอปนี้เป็นเพียงค่าเริ่มต้นทั่วไปตามหลักการของ
> พ.ร.บ.คุ้มครองแรงงาน สำหรับลูกจ้างรายเดือนเท่านั้น **ไม่ใช่คำแนะนำทางกฎหมายหรือบัญชีเงินเดือน**
> กรุณาตรวจสอบกับฝ่ายบุคคล/นักบัญชี/ที่ปรึกษากฎหมายแรงงานของบริษัทก่อนนำตัวเลขจากระบบนี้ไปใช้
> จ่ายค่าจ้างจริง และปรับค่าต่างๆ ใน `js/config.js` (`OT_RULES`) ให้ตรงกับนโยบายบริษัท/สัญญาจ้างจริง

## ขั้นตอนที่ 1: สร้างโปรเจกต์ Firebase (ฟรี ไม่ต้องผูกบัตร)

> แนะนำให้สร้าง Firebase project **แยกต่างหาก** จากแอปอื่นๆ ที่คุณมีอยู่ (เช่นแอปแจ้งซ่อม)
> เพื่อไม่ให้ข้อมูลพนักงาน/เวลาทำงาน ปะปนกับข้อมูลระบบอื่น

1. ไปที่ https://console.firebase.google.com → **สร้างโปรเจกต์ (Add project)**
2. ตั้งชื่อโปรเจกต์ เช่น `hr-attendance-app` → กด Create
3. ในเมนูซ้าย เปิดใช้งาน **Firestore Database** → Create database → เลือกโหมด Production → เลือก location (เช่น `asia-southeast1`) — อยู่ในแผนฟรี Spark ไม่ต้องผูกบัตร
4. ไปที่ **Project settings (ไอคอนเฟือง) > General** → เลื่อนลงไปที่ "Your apps" → กดไอคอน **</>** (Web app) → ตั้งชื่อ (เช่น "hr-web") → Register app
5. คัดลอกค่า `firebaseConfig` ที่ได้ (apiKey, authDomain, projectId, storageBucket, messagingSenderId, appId) ไปใส่ในไฟล์ **`js/config.js`** ส่วน `FIREBASE_CONFIG`

### ตั้งค่ากฎความปลอดภัยของฐานข้อมูล (สำคัญ — ต้องทำ ไม่งั้นแอปใช้งานไม่ได้)
1. ไปที่เมนู **Firestore Database > กฎ (Rules)** ในโปรเจกต์ Firebase ของคุณ
2. เปิดไฟล์ **`firestore.rules`** ที่แนบมาในซิปนี้ → คัดลอกเนื้อหาทั้งหมด
3. วางแทนที่เนื้อหาเดิมในกล่องกฎบนเว็บ Firebase Console → กด **Publish**

## ขั้นตอนที่ 2: กรอกข้อมูลบริษัทเริ่มต้นใน `js/config.js`

เปิดไฟล์ `js/config.js` แล้วแก้ไขส่วนต่างๆ ตามบริษัทจริง:

- `COMPANY` — ชื่อบริษัท/โลโก้
- `ADMINS` — รายชื่อแอดมิน **4 คนที่ได้รับอนุมัติ** พร้อมรหัส PIN ของแต่ละคน (ดูหัวข้อ "การระบุตัวตน" ด้านล่าง)
- `DEPARTMENTS` — รายชื่อแผนกในบริษัท (ตั้งตามโครงสร้างจริง 7 แผนก: ผู้บริหาร, Accounting, Architect, Build-In, Marketing, Purchasing, Supervisor all team)
- `SHIFTS` — กะการทำงานเริ่มต้น 4 กะ (แก้ไข/เพิ่ม/ลบเพิ่มเติมได้ที่หน้าแอดมินหลัง deploy แล้วเช่นกัน)
- `HOLIDAYS_2026` — วันหยุดประจำปี (คัดลอกมาจากปฏิทินที่แนบให้ตอนแรกแล้ว แก้ไขเพิ่มเติมได้ที่หน้าแอดมิน)
- `EMPLOYEES_SEED` — รายชื่อพนักงานเริ่มต้น **36 คน แยกตามแผนก** (คัดลอกมาจากไฟล์รายชื่อพนักงานที่แนบให้)
  ⚠️ ใช้ตอนหว่านเมล็ดข้อมูลครั้งแรกเท่านั้น — ปัจจุบันพนักงานลงทะเบียนด้วยข้อมูลตัวเองแทน (ดูหัวข้อ
  "การระบุตัวตน" ด้านล่าง) รายชื่อชุดนี้จึงเป็นแค่ตัวอย่าง/ค่าเริ่มต้น ไม่ได้ใช้เป็นลิสต์ให้เลือกอีกแล้ว
- `OT_RULES` — อัตรา OT ต่างๆ (ดูคำเตือนด้านบนสุดของไฟล์นี้)

ค่าเริ่มต้นของ `SHIFTS`, `HOLIDAYS_2026` และ `EMPLOYEES_SEED` จะถูก "หว่านเมล็ด" (seed) เข้า Firestore
ให้อัตโนมัติในการเปิดแอป **หน้าไหนก็ได้เป็นครั้งแรก** (ทั้ง `index.html` และ `admin.html` ต่างก็ตรวจสอบ
และหว่านเมล็ดให้เหมือนกัน เฉพาะ collection ที่ยังไม่มีข้อมูลเลยเท่านั้น) ไม่จำเป็นต้องเปิดหน้าแอดมินก่อน
หลังจากหว่านเมล็ดครั้งแรกแล้ว ให้ไปแก้ไขที่หน้าแอดมินแทนการแก้ไฟล์นี้ซ้ำ เพื่อไม่ต้อง deploy ใหม่ทุกครั้ง
ที่มีการเปลี่ยนกะ/วันหยุด/พนักงาน

> ⚠️ **ถ้าเปิดแอปแล้วเจอ "ไม่พบรายชื่อพนักงาน กรุณาติดต่อฝ่ายบุคคล"** แปลว่ายังไม่ได้กรอกค่า
> `FIREBASE_CONFIG` ให้ครบ (ยังเป็น `REPLACE_ME` อยู่) หรือยังไม่ได้ publish `firestore.rules` —
> ทำสองขั้นตอนนี้ให้เสร็จก่อน (ขั้นตอนที่ 1 ด้านบน) แล้วรีเฟรชหน้าเว็บใหม่ ระบบจะหว่านเมล็ดข้อมูลให้เอง

> พนักงานทั้ง 36 คนที่หว่านเมล็ดมาให้ จะถูกตั้งกะเริ่มต้นเป็น "กะออฟฟิศ" และวันหยุดประจำสัปดาห์เป็น
> วันอาทิตย์ให้ทุกคนไว้ก่อน ไปปรับกะ/วันหยุดจริงของแต่ละคนที่หน้าแอดมิน → แท็บ "พนักงาน" ได้ทีหลัง

## ขั้นตอนที่ 4: สร้าง LINE LIFF App (ทำให้เป็น LINE Mini App) — ต้องทำเองผ่านบัญชี LINE ของคุณ

1. ไปที่ https://developers.line.biz/console/ → ล็อกอินด้วยบัญชี LINE ของคุณ (หรือของบริษัท)
2. สร้าง **Provider** (ถ้ายังไม่มี) → ภายใต้ Provider นั้น **Create a new channel** → เลือกประเภท **LINE Login**
3. เข้าไปที่ Channel ที่สร้าง → แท็บ **LIFF** → กด **Add**
   - LIFF app name: ระบบลงเวลาทำงาน
   - Size: **Full**
   - Endpoint URL: ลิงก์ที่ได้จากขั้นตอนที่ 3 เช่น `https://random-name-12345.netlify.app`
   - Scope: ติ๊ก **profile** และ **openid** ก็พอ (ไม่ต้องติ๊ก `chat_message.write` — แอปนี้ไม่ใช้
     `liff.sendMessages()` ส่งข้อความแจ้งเตือนแล้ว การแจ้งเตือนทั้งหมดส่งจากฝั่งเซิร์ฟเวอร์ผ่าน Cloud
     Function แทน ดูหัวข้อ "แจ้งเตือนเข้า LINE ทันทีที่เช็คอิน/เช็คเอาท์ และยื่น/พิจารณาคำขอลา" ด้านล่าง)
   - Bot link feature: เลือก Off ได้ถ้ายังไม่มี LINE OA
   - กด Add
4. คัดลอกค่า **LIFF ID** ที่ได้ → ใส่ในไฟล์ `js/config.js`:
   ```js
   export const LIFF_ID = "1234567890-AbCdEfGh"; // ← LIFF ของหน้าพนักงาน (index.html)
   ```
5. ทำซ้ำขั้นตอน 3-4 อีกครั้งเพื่อสร้าง **LIFF app ที่สอง** สำหรับหน้าแอดมิน โดยตั้ง Endpoint URL เป็น
   `https://your-site.netlify.app/admin.html` (Scope เหมือนกัน: **profile** + **openid** พอ)
   แล้วนำ LIFF ID ที่ได้ไปใส่:
   ```js
   export const LIFF_ID_ADMIN = "1234567890-XxYyZzWw"; // ← LIFF ของหน้าแอดมิน (admin.html)
   ```
6. อัปโหลดไฟล์ที่แก้แล้วขึ้น hosting เดิมอีกครั้ง
7. เปิดลิงก์ผ่านแอป LINE (ส่งลิงก์ไปแชทตัวเอง/กลุ่ม แล้วกดเปิด) — พนักงานแต่ละคนจะเจอหน้า
   "ผูกบัญชี LINE ของคุณ" ให้เลือกชื่อตัวเอง **ครั้งเดียว** หลังจากนั้นระบบจะจำบัญชี LINE นั้นได้อัตโนมัติ
   ทุกครั้งที่เปิดผ่าน LINE ไม่ต้องเลือกชื่อซ้ำอีก แม้เปลี่ยนเครื่อง (ดูหัวข้อ "การระบุตัวตน" ด้านล่าง)

> หากยังไม่พร้อมทำ LIFF ตอนนี้ ปล่อย `LIFF_ID` / `LIFF_ID_ADMIN` เป็น `""` ไว้ได้ แอปจะทำงานเป็น
> เว็บแอปปกติ เปิดผ่านเบราว์เซอร์ได้ทันทีโดยไม่มีผลกระทบใดๆ (จะกลับไปใช้วิธีเลือกชื่อ+จดจำในอุปกรณ์แทน)

> ℹ️ **เรื่องการแจ้งเตือนเข้า LINE:** แอปรุ่นนี้ **ไม่ใช้** `liff.sendMessages()` แล้ว (เดิมฟังก์ชันนี้จะ
> โพสต์ข้อความ "เข้าห้องแชทที่เปิดแอปอยู่ตอนนั้น" ซึ่งอาจเป็นห้องแชทกลุ่มโดยไม่ตั้งใจ ทำให้ข้อความหลุดไป
> โผล่ที่อื่นนอกเหนือจากแชท 1:1 กับ LINE OA) ตอนนี้การแจ้งเตือนทุกจุด (เช็คอิน/เช็คเอาท์, ยื่น/พิจารณา
> คำขอลา, ส่งสรุปประจำวันของแอดมิน) ส่งผ่าน **LINE Messaging API แบบ push message จากฝั่งเซิร์ฟเวอร์
> (Netlify Function — ฟรี ไม่ต้องผูกบัตรเครดิต/ไม่ต้องอัปเกรดแผน Firebase)** เท่านั้น จึงส่งเข้า
> **แชทส่วนตัว 1:1 ระหว่างผู้รับกับ LINE OA เท่านั้น** รับประกันว่าจะไม่หลุดไปที่ห้องแชทกลุ่มหรือที่อื่น —
> ต้องตั้งค่าเพิ่มเล็กน้อย (แค่ตั้งค่า Channel Access Token ที่ Netlify ไม่มีขั้นตอนผูกบัตรใดๆ) ดูขั้นตอน
> เต็มที่หัวข้อ "แจ้งเตือนเข้า LINE ทันทีที่เช็คอิน/เช็คเอาท์ และยื่น/พิจารณาคำขอลา" ด้านล่าง

## ขั้นตอนที่ 5: เชื่อมกับ LINE Official Account — ปุ่ม Check In-Out / ลางาน / สรุปการทำงาน

ทำให้พนักงานเปิดแอปนี้ได้จาก **เมนูด้านล่างของแชท LINE Official Account (Rich Menu)** โดยไม่ต้องหาลิงก์เอง
วิธีนี้**ไม่ต้องมีเซิร์ฟเวอร์เพิ่ม ไม่มีค่าใช้จ่าย** เพราะปุ่มแต่ละปุ่มแค่เปิด LIFF URL เดิม (จากขั้นตอนที่ 4)
พร้อมพารามิเตอร์บอกว่าจะให้เปิดไปที่แท็บไหน — โค้ดฝั่ง `index.html` รองรับพารามิเตอร์นี้ไว้ให้แล้ว

1. เชื่อม LIFF app (จากขั้นตอนที่ 4) เข้ากับ LINE Official Account ที่มีอยู่แล้ว: ใน LINE Developers Console
   ไปที่ Channel ของ LIFF app (ฝั่งพนักงาน) → แท็บ **LIFF** → ตรวจสอบว่า Provider เดียวกันกับ Messaging API
   Channel ของ LINE OA ที่ใช้อยู่ (ถ้าคนละ Provider ให้ย้ายหรือสร้าง LIFF ใหม่ภายใต้ Provider เดียวกับ OA)
2. คัดลอก LIFF URL เต็ม จาก LINE Developers Console (รูปแบบ `https://liff.line.me/xxxxxxxxxx-xxxxxxxx`)
   แล้วต่อพารามิเตอร์ `?tab=` ท้ายลิงก์ตามฟังก์ชันที่ต้องการ:
   - Check In-Out: `https://liff.line.me/xxxxxxxxxx-xxxxxxxx?tab=home`
   - ลางาน: `https://liff.line.me/xxxxxxxxxx-xxxxxxxx?tab=leave`
   - สรุปการทำงาน (ประวัติ): `https://liff.line.me/xxxxxxxxxx-xxxxxxxx?tab=history`
3. ไปที่ **LINE Official Account Manager** (https://manager.line.biz/) → เลือก OA ของบริษัท →
   เมนู **Rich menu** → **Create a rich menu**
   - ออกแบบภาพเมนู (แนะนำแบ่ง 3 ช่องเท่าๆ กัน หรือใช้เทมเพลตสำเร็จรูปที่ให้เลือก)
   - ตั้งค่าแต่ละช่อง (Tap area) เป็น Action ประเภท **Link** แล้ววางลิงก์จากข้อ 2 ตามฟังก์ชันนั้นๆ
     (ช่องที่ 1 → ลิงก์ `?tab=home`, ช่องที่ 2 → ลิงก์ `?tab=leave`, ช่องที่ 3 → ลิงก์ `?tab=history`)
   - กด **Publish**
4. ทดสอบ: เปิดแชท LINE OA ในมือถือ จะเห็นเมนูปุ่มด้านล่างแชท กดแต่ละปุ่มแล้วแอปควรเปิดไปที่แท็บที่กำหนดทันที

> ขั้นตอนนี้ (สร้าง LIFF/Rich Menu) ต้องทำผ่านบัญชี LINE ของบริษัทเอง (ต้องล็อกอินด้วย LINE/สแกน QR
> จากมือถือ) จึงไม่สามารถทำแทนให้ได้ทั้งหมด — ทำตามขั้นตอนข้างบนได้เลย ถ้าติดขั้นตอนไหนแจ้งมาได้

## การระบุตัวตน (ไม่มีระบบล็อกอินจริง)

### ฝั่งพนักงาน — ลงทะเบียนด้วยข้อมูลตัวเองครั้งแรก (ไม่ใช่การเลือกชื่อจากลิสต์อีกต่อไป)

พนักงานทุกคน **กรอกข้อมูลของตัวเองครั้งแรกที่เปิดแอป** เท่านั้น (ครั้งเดียวตลอดไป) — **ชื่อ-นามสกุล**,
**ชื่อเล่น**, และ **วันเดือนปีเกิด** ระบบจะสร้างพนักงานใหม่ในฐานข้อมูลจากข้อมูลที่กรอกทันที ไม่ต้องมี
ใครมาเตรียมรายชื่อไว้ล่วงหน้า (แก้ปัญหาเดิมที่ "รายชื่อมาไม่ครบ" เพราะระบบเก่าต้องเลือกจากลิสต์ที่แอดมิน
ใส่ไว้ล่วงหน้าเท่านั้น) และทันทีที่ลงทะเบียนเสร็จ **ข้อมูลจะถูก "ล็อก" ไว้กับบัญชี LINE หรืออุปกรณ์ที่
ลงทะเบียนเท่านั้น** ทำให้คนอื่นไม่สามารถสวมรอยเข้าระบบโดยใช้ชื่อเดียวกันได้อีก (ไม่ว่าจะเปิดผ่าน LINE
หรือเบราว์เซอร์ปกติก็ตาม)

- **เปิดผ่านแอป LINE (แนะนำ ถ้าตั้งค่า `LIFF_ID` ไว้แล้ว):** ระบบจะดึงบัญชี LINE ของผู้ใช้มาตรวจสอบ
  อัตโนมัติ ถ้าเคยลงทะเบียนไว้แล้วจะเข้าระบบทันทีโดย **ไม่ต้องกรอกข้อมูลซ้ำ** (เหมือน Hello Boss) ไม่ว่า
  จะเปลี่ยนไปเปิดจากเครื่องไหนก็ตาม (เพราะผูกกับบัญชี LINE ไม่ใช่อุปกรณ์) ถ้ายังไม่เคยลงทะเบียน จะให้
  กรอกฟอร์มลงทะเบียน แล้วผูกกับบัญชี LINE นั้นทันที

- **เปิดผ่านเบราว์เซอร์ปกติ (ยังไม่ได้ตั้งค่า `LIFF_ID` หรือเปิดนอกแอป LINE):** หลังลงทะเบียนครั้งแรก
  ข้อมูลจะถูกผูกกับ **อุปกรณ์เครื่องนี้โดยเฉพาะ** (เก็บเป็นรหัสอุปกรณ์สุ่มถาวรไว้ใน localStorage และ
  บันทึกคู่กับพนักงานคนนั้นไว้ใน Firestore) เปิดครั้งถัดไปจากเครื่องเดิมจะเข้าระบบอัตโนมัติไม่ต้องกรอกซ้ำ

- **ลงทะเบียนผิดพลาด/ต้องการรีเซ็ต:** แอดมินกด **"ยกเลิกการลงทะเบียน"** ให้ที่หน้าแอดมิน → แท็บ
  "พนักงาน" (ปลดล็อกได้ทั้งกรณีผูกกับ LINE และผูกกับอุปกรณ์) แล้วให้พนักงานคนนั้นลงทะเบียนใหม่อีกครั้ง
  (จะกลายเป็นพนักงานคนเดิม ไม่ได้สร้างซ้ำ เพราะใช้ข้อมูลพนักงานเดิมที่มีอยู่แล้ว)

> ℹ️ พนักงานที่เคยมีอยู่ก่อนอัปเดตนี้ (ลงทะเบียนไปแล้วจากระบบเลือกชื่อแบบเดิม) จะยังใช้งานได้ตามปกติ
> ไม่กระทบใดๆ — ส่วนพนักงานที่ **ยังไม่เคยลงทะเบียน** จากรายชื่อชุดเดิม (36 คนที่หว่านเมล็ดไว้ตอนแรก)
> จะไม่ถูกใช้งานอีกต่อไป แนะนำให้แอดมินไปที่แท็บ "พนักงาน" แล้วกด "ปิดใช้งาน" รายชื่อเก่าที่ไม่มีใครใช้
> ทิ้งไว้ เพื่อไม่ให้ปนกับพนักงานที่ลงทะเบียนเข้ามาใหม่ด้วยตัวเอง

> ⚠️ กลไกนี้ป้องกัน "คนอื่นสวมรอยเข้าไปเช็คอิน/ดูข้อมูลแทนกัน" ได้ในระดับที่เหมาะกับการใช้งานภายใน
> บริษัท แต่ยังไม่ใช่การยืนยันตัวตนแบบเข้ารหัส (เช่น รหัสผ่าน/OTP) เทียบเท่า Firebase Authentication —
> ระบบมีการบังคับถ่ายรูปยืนยันตัวตนทุกครั้งที่เช็คอิน/เช็คเอาท์เพิ่มอีกชั้นหนึ่ง (ดูหัวข้อด้านล่าง) ถ้า
> ต้องการความปลอดภัยสูงขึ้นอีกขั้นแจ้งมาได้เสมอ

- **ฝั่งแอดมิน:** จำกัดสิทธิ์ไว้เฉพาะ **4 คนที่ได้รับอนุมัติ** (ตามรายชื่อ+รหัส PIN ใน `ADMINS` ของ
  `js/config.js`) เมื่อเลือกชื่อแล้วต้อง **กรอกรหัส PIN ให้ถูกต้อง** ก่อนถึงจะเข้าหน้าแดชบอร์ดได้
  ค่าเริ่มต้นตั้งไว้ตามที่อนุมัติดังนี้ (แก้ไข/เพิ่ม/ลบคนและ PIN ได้ที่ `js/config.js` → `ADMINS`):

  | ชื่อ | PIN เริ่มต้น |
  |---|---|
  | K.Eddie | 12011976 |
  | K.Peggy | 11071976 |
  | Nok | 09091994 |
  | Treeya | 17082542 |

  หลังกรอก PIN ถูกต้องครั้งแรก ระบบจะจดจำไว้ในอุปกรณ์เครื่องนั้น (เหมือนฝั่งพนักงาน) ไม่ต้องกรอกซ้ำ
  ทุกครั้งที่เปิดแอป จนกว่าจะกด "เปลี่ยนผู้ใช้งาน" หรือล้างข้อมูลเบราว์เซอร์

> ⚠️ **ข้อควรระวังด้านความปลอดภัย:** รหัส PIN นี้เป็นเพียง **ชั้นป้องกันเบื้องต้น** (ตรวจสอบง่ายๆ
> ในเบราว์เซอร์ฝั่งผู้ใช้) ไม่ใช่ระบบล็อกอินที่เข้ารหัส/ปลอดภัยเทียบเท่า Firebase Authentication —
> ป้องกันคนทั่วไปที่ไม่รู้ PIN ไม่ให้กดเข้าหน้าแอดมินได้ง่ายๆ แต่ไม่ควรใช้เป็นเกราะป้องกันข้อมูล
> ที่ละเอียดอ่อนมาก (เช่น เงินเดือน) ถ้าต้องการความปลอดภัยสูงขึ้นในอนาคต แนะนำให้เพิ่ม Firebase
> Authentication (ล็อกอินด้วยอีเมล/รหัสผ่านหรือเบอร์โทร) แจ้งมาได้เสมอ นอกจากนี้ควรส่งลิงก์แอปนี้
> ให้เฉพาะพนักงาน/ทีมงานที่เกี่ยวข้องเท่านั้น อย่าเผยแพร่ต่อสาธารณะ

## หลักการคำนวณ OT

สูตรที่ใช้ (`js/ot-calc.js`, มีชุดทดสอบยืนยันความถูกต้องใน `js/ot-calc.test.mjs`):

- **วันทำงานปกติ:** ชั่วโมงทำงานภายใน `normalWorkMinutesPerDay` (ค่าเริ่มต้น 8 ชม.) ถือว่ารวมอยู่ใน
  เงินเดือนแล้ว ส่วนที่เกินคิดเป็น **OT อัตรา 1.5 เท่า** (`otRateWeekday`)
- **วันหยุดประจำสัปดาห์ / วันหยุดนักขัตฤกษ์:** ชั่วโมงทำงานภายในเกณฑ์ปกติ จ่ายเพิ่มที่อัตรา
  **1 เท่า** (`holidayBaseRate`) ส่วนที่เกินคิดเป็น **OT อัตรา 3 เท่า** (`holidayOtRate`)
- **มาสาย:** นับว่า "สาย" เมื่อเช็คอินช้ากว่าเวลาเริ่มกะเกิน `lateThresholdMinutes` นาที (ค่าเริ่มต้น 5 นาที)
- **เช็คอิน/เอาท์หลายรอบต่อวัน:** เวลาที่ "หายไป" ระหว่างรอบ (เช่น ออกไปทานข้าวเที่ยง) จะไม่ถูกนับ
  เป็นเวลาทำงานโดยอัตโนมัติ เพราะระบบคิดจากผลรวมของแต่ละรอบเช็คอิน-เอาท์จริงเท่านั้น
- **กะข้ามเที่ยงคืน (เช่น กะดึก 21:00-06:00):** ระบบจะรวมเวลาทำงานให้ถูกต้องข้ามวันปฏิทิน และนับ
  เป็น "วันกะ" เดียวกับวันที่กะเริ่มต้น (เช่น เข้างาน 21:00 คืนวันจันทร์ ออกงาน 06:00 เช้าวันอังคาร
  จะถูกนับรวมเป็นข้อมูลของ "วันจันทร์")

ปรับตัวเลขทั้งหมดนี้ได้ที่ `OT_RULES` ใน `js/config.js`

## การสลับวันหยุด (Day-off Swap)

เมื่อพนักงานต้องมาทำงานในวันหยุดของตัวเอง แล้วขอหยุดชดเชยในวันอื่นแทน ทำได้ 2 ทาง:

1. **พนักงานยื่นขอเอง** — ที่หน้าแอป → แท็บ "การลา" → การ์ด "🔄 ขอสลับวันหยุด" ระบุวันหยุดเดิม/วันที่
   ขอหยุดแทน/เหตุผล แล้วกดส่ง คำขอจะอยู่ในสถานะ **"รออนุมัติ"** และยังไม่มีผลต่อปฏิทินจนกว่าแอดมินจะ
   กดอนุมัติที่หน้าแอดมิน → แท็บ "กะ/วันหยุด/สลับวันหยุด" (การ์ด "🔄 สลับวันหยุด" จะขึ้นรายการที่รอ
   อนุมัติไว้ด้านบนสุด พร้อมปุ่ม "✅ อนุมัติ" / "❌ ไม่อนุมัติ")
2. **แอดมินบันทึกให้โดยตรง** — กรอกฟอร์มเดียวกันที่หน้าแอดมิน จะถือว่า **อนุมัติทันที** ไม่ต้องรอ

ไม่ว่าจะยื่นทางไหน ผลลัพธ์เมื่ออนุมัติแล้วเหมือนกัน:

- **วันหยุดเดิม** (วันที่เดิมเป็นวันหยุด แต่จะมาทำงานแทน) → ระบบจะถือว่าวันนี้เป็น **วันทำงานปกติ**
  (คิด OT แบบวันทำงานปกติ 1.5x ไม่ใช่อัตราวันหยุด 3x)
- **วันหยุดใหม่** (วันที่เดิมเป็นวันทำงาน แต่จะใช้เป็นวันหยุดแทน) → ระบบจะถือว่าวันนี้เป็น **วันหยุด**

หลังอนุมัติแล้ว ระบบจะนำไปใช้คำนวณ OT และแสดงผลในสรุปประจำวัน/ประวัติ/รายงานของพนักงานคนนั้น
โดยอัตโนมัติ ไม่ต้องตั้งค่าอะไรเพิ่มอีก

## โควต้าวันลา + สรุปการใช้วันลารายบุคคล

ระบบติดตามโควต้าวันลา **3 ประเภท**: ลาป่วย, ลากิจ, ลาพักร้อน (ไม่รวมลาคลอด/ลาไม่รับค่าจ้าง/อื่นๆ
เพราะปกติไม่มีโควต้าตายตัว) แบบ **รายปีปฏิทิน** (นับ 1 ม.ค. - 31 ธ.ค. ของแต่ละปี ตามวันที่เริ่มลาของ
คำขอ) รีเซ็ตยอดใหม่อัตโนมัติทุกวันที่ 1 มกราคม โดยไม่ต้องมีใครไปกดรีเซ็ตเอง (เพราะระบบคำนวณสดจากปีของ
วันที่ในคำขอลาที่อนุมัติแล้วเสมอ ไม่ได้เก็บยอดคงเหลือเป็นตัวเลขนิ่งๆ ไว้ที่ไหน)

### ตั้งค่าโควต้าเริ่มต้นของบริษัท

หน้าแอดมิน → แท็บ "📊 สรุปวันลา" → การ์ดด้านบน "⚙️ ตั้งค่าโควต้าวันลาเริ่มต้น" กรอกจำนวนวัน/ปีของแต่ละ
ประเภท แล้วกด **"💾 บันทึกค่าเริ่มต้น"** (ค่านี้จะถูกใช้เป็นค่าเริ่มต้นให้พนักงานที่เพิ่มใหม่ทุกคนนับจาก
นี้) ค่าเริ่มต้นในโค้ด (`js/config.js` → `DEFAULT_LEAVE_QUOTA`) คือ **ลาป่วย 30 / ลากิจ 3 / ลาพักร้อน 6
วัน/ปี** ตามนโยบายบริษัท ปรับได้จากหน้านี้โดยไม่ต้องแก้โค้ด

ถ้าต้องการ "รีเซ็ต" ให้พนักงานทุกคนกลับไปใช้ค่าเริ่มต้นเดียวกันหมด (เขียนทับค่าที่เคยปรับเฉพาะรายคนไว้)
กดปุ่ม **"📋 ใช้ค่านี้กับพนักงานทุกคน"** (ระบบจะถามยืนยันก่อนเพราะเป็นการเขียนทับ)

### ปรับโควต้าเฉพาะบุคคล (ถ้าบางคนไม่เหมือนค่ามาตรฐาน)

หน้าแอดมิน → แท็บ "จัดการพนักงาน" → แก้ไขพนักงานคนนั้น → เลื่อนลงมาที่ "โควต้าวันลาต่อปีของพนักงานคนนี้"
ปรับตัวเลขแล้วกดบันทึก ช่องนี้จะโชว์ค่าที่ใช้งานจริงของคนนั้น (ถ้ายังไม่เคยปรับ จะโชว์ค่าเริ่มต้นของ
บริษัทให้ดูก่อน)

### การหักโควต้า + สรุปยอดคงเหลือ

- หักโควต้าเฉพาะคำขอลาที่ **"อนุมัติแล้ว"** เท่านั้น (คำขอที่ "รออนุมัติ" หรือ "ไม่อนุมัติ" ไม่มีผลต่อ
  ยอดคงเหลือ)
- ฝั่งพนักงาน: แท็บ "การลา" จะมีการ์ด "📊 สรุปวันลาปีนี้" แสดงยอด ใช้ไป/โควต้า/คงเหลือ ของทั้ง 3
  ประเภท พร้อมแถบสีบอกสัดส่วนที่ใช้ไปแล้ว อัปเดตอัตโนมัติทุกครั้งที่มีการยื่น/แอดมินอนุมัติคำขอลาใหม่
- ฝั่งแอดมิน: แท็บ "📊 สรุปวันลา" มีตารางสรุปยอด ใช้ไป/โควต้า/คงเหลือ ของพนักงานทุกคนในปีปัจจุบัน
  ดูภาพรวมทั้งบริษัทได้ในหน้าเดียว

### ยื่นเกินโควต้าได้ไหม?

**ได้** — ระบบ**ไม่บล็อก**การยื่นคำขอลาแม้จะเกินโควต้าที่เหลืออยู่ (ตามนโยบายบริษัทที่อยากให้ยื่นได้ตาม
ปกติก่อน ค่อยพิจารณาเป็นกรณีไป) แต่เมื่อแอดมินเปิดดูรายการคำขอที่ "รออนุมัติ" ในแท็บ "คำขอลา" ระบบจะ
**ขึ้นข้อความเตือนสีแดง (⚠️)** ใต้คำขอนั้นทันที บอกว่าใช้โควต้าไปเท่าไหร่แล้ว และถ้าอนุมัติคำขอนี้จะเกิน
โควต้าไปกี่วัน เพื่อให้แอดมินใช้ดุลยพินิจตอนกดอนุมัติ/ไม่อนุมัติ

## กะการทำงาน (แอดมินกำหนด) + ถ่ายรูป/ตำแหน่งยืนยันตัวตน

- **กะกำหนดโดยแอดมิน:** พนักงานแต่ละคนมีกะประจำ (`shiftId`) ที่แอดมินตั้งไว้ล่วงหน้าในหน้าจัดการ
  พนักงาน (`admin.html` → แก้ไขพนักงาน → เลือกกะ) พนักงานไม่ต้องเลือกกะเองทุกวัน — ระบบจะดึงกะที่
  แอดมินตั้งไว้มาใช้คำนวณเวลาเข้างาน/มาสาย/OT ให้อัตโนมัติทันทีที่กดเช็คอิน/เช็คเอาท์
- **บังคับถ่ายรูปยืนยันตัวตนทุกครั้ง:** ทุกครั้งที่กดเช็คอิน/เช็คเอาท์ ระบบจะเปิดกล้องหน้าให้ถ่ายรูป
  ก่อนเสมอ (ถ้ายกเลิกการถ่ายรูปจะไม่มีการบันทึกเวลา) รูปจะถูกบีบอัดในเบราว์เซอร์ก่อนบันทึกเพื่อประหยัด
  พื้นที่ ดูรูปย้อนหลังได้จากไอคอน 📷 ในสรุปวันนี้ (ฝั่งพนักงาน) หรือคอลัมน์ "รูปยืนยันตัวตน" ในตาราง
  รายงาน (ฝั่งแอดมิน)
- **บังคับต้องมีตำแหน่ง GPS ทุกครั้ง:** ทุกครั้งที่เช็คอิน/เช็คเอาท์ ระบบจะขอพิกัดตำแหน่งปัจจุบันคู่กับ
  รูปยืนยันตัวตน — ⚠️ ถ้าขอพิกัดไม่สำเร็จ (ผู้ใช้ไม่อนุญาต/ปิดสิทธิ์ตำแหน่ง/สัญญาณ GPS อ่อนเกินไป) ระบบ
  จะ **ยกเลิกการบันทึกเวลาทั้งหมด** (ไม่บันทึกทั้งเวลาและรูปที่เพิ่งถ่ายไป) พร้อมแจ้งเหตุผลและวิธีแก้ให้
  ชัดเจน (เช่น ให้ไปเปิดสิทธิ์ตำแหน่งในตั้งค่าเบราว์เซอร์/แอป LINE ก่อน) แล้วให้กดปุ่มเช็คอิน/เช็คเอาท์
  ใหม่อีกครั้ง — ระบบจะเริ่มขอพิกัด GPS "คู่ขนาน" ไปกับตอนที่ผู้ใช้กำลังถ่ายรูป (ไม่ใช่รอถ่ายรูปเสร็จ
  ค่อยขอทีหลัง) เพื่อลดเวลารอรวมให้สั้นที่สุด (timeout 15 วินาที)
- **ป้องกันการเช็คอินค้าง:** ทุกขั้นตอนที่อาจใช้เวลานาน (ถอดรหัส/บีบอัดรูป, ขอพิกัด GPS, บันทึกลง
  ฐานข้อมูล) มีการจำกัดเวลาสูงสุดที่ยอมรอไว้ทุกจุด ถ้าขั้นตอนไหนช้าผิดปกติ (เน็ตไม่เสถียร/เครื่องช้า)
  ระบบจะแจ้งเตือนข้อผิดพลาดให้กดลองใหม่ แทนที่จะค้างหน้าจอ "กำลังประมวลผล" ไปเรื่อยๆ แบบไม่รู้จบ

## แยกแผนก + หัวหน้าทีม + แจ้งเตือน LINE เมื่อทีมเช็คอิน/เช็คเอาท์

ฟีเจอร์นี้มี 2 ส่วนที่ **พร้อมใช้งานทันที** (ไม่ต้องตั้งค่าอะไรเพิ่ม) และ 1 ส่วนที่ **ต้องตั้งค่า
เพิ่มเติมเอง** (เพราะต้องมีเซิร์ฟเวอร์ยิงข้อความออกไปหา LINE ซึ่งต้องอัปเกรด Firebase)

### ✅ ใช้งานได้ทันที ไม่ต้องตั้งค่าเพิ่ม

1. **กรองรายงานตามแผนก** — หน้าแอดมิน → แท็บ "รายงาน/OT" → เลือกแผนกจากดรอปดาวน์ "ทุกแผนก" ที่เพิ่ม
   เข้ามาใหม่ (อยู่ก่อนดรอปดาวน์ "พนักงานทั้งหมด") ระบบจะกรองทั้งตารางรายงานและดรอปดาวน์รายชื่อ
   พนักงานให้เหลือเฉพาะคนในแผนกนั้นทันที
2. **แต่งตั้งหัวหน้าทีม** — หน้าแอดมิน → แท็บ "พนักงาน" → แก้ไขพนักงานที่ต้องการ → เลือกแผนกของเขาไว้
   ก่อน แล้วติ๊ก "แต่งตั้งเป็นหัวหน้าทีมของแผนกนี้" → บันทึก จะเห็นป้าย "⭐ หัวหน้าทีม" ในรายชื่อ
   พนักงาน (แผนกหนึ่งตั้งหัวหน้าทีมได้มากกว่า 1 คน)
3. **แท็บ "ทีมของฉัน" (ดูอย่างเดียว)** — พนักงานที่ถูกตั้งเป็นหัวหน้าทีมจะเห็นแท็บใหม่ "👥 ทีมของฉัน"
   ในหน้าแอปของตัวเอง (`index.html`) แสดงสถานะเช็คอิน/เช็คเอาท์วันนี้ (หรือย้อนหลังเลือกวันที่ได้) ของ
   ทุกคนในแผนกเดียวกัน พร้อมเวลาเข้า-ออก ตำแหน่ง GPS และรูปยืนยันตัวตน — เป็นสิทธิ์แบบ "ดูอย่างเดียว
   เฉพาะทีมตัวเอง" เท่านั้น แก้ไขข้อมูลอะไรไม่ได้จากแท็บนี้

### ⚙️ ต้องตั้งค่าเพิ่ม (ฟรี 100% ไม่ต้องผูกบัตรเครดิต/ไม่มีค่าใช้จ่ายเพิ่ม): แจ้งเตือนเข้า LINE ทันทีที่เช็คอิน/เช็คเอาท์ และยื่น/พิจารณาคำขอลา

ส่วนนี้ทำให้ระบบส่งข้อความ LINE แจ้งเตือนทันทีครอบคลุม 3 เหตุการณ์อัตโนมัติ (ไม่ต้องกดอะไรเอง) และยังเป็น
กลไกเดียวกันที่ปุ่ม "📤 ส่งสรุปวันนี้เข้า LINE" ของแอดมิน (กดเอง) ใช้ส่งข้อความด้วย

> ✅ **ไม่ต้องอัปเกรด Firebase เป็นแผน Blaze และไม่ต้องผูกบัตรเครดิตใดๆ ทั้งสิ้น** ส่วนนี้ใช้
> **Netlify Functions** (ฟีเจอร์ฟรีของ Netlify ที่คุณใช้ deploy เว็บอยู่แล้ว — ให้โควตาฟรี 125,000
> ครั้ง/เดือน ไม่ต้องผูกบัตรถึงจะใช้ได้) เป็นเซิร์ฟเวอร์กลางเก็บ LINE Channel Access Token แทน Firebase
> Cloud Functions ทำงานแบบเดียวกันทุกประการ (ส่งข้อความเข้าแชทส่วนตัว 1:1 กับ LINE OA เท่านั้น ไม่มีทาง
> หลุดไปห้องแชทกลุ่ม) เพียงแต่ไม่มีค่าใช้จ่ายหรือความเสี่ยงเรื่องบัตรเครดิตเลย

ทั้ง 4 กรณีที่ส่งแจ้งเตือน:

1. **เช็คอิน/เช็คเอาท์** — แจ้งเตือน (ก) ตัวพนักงานเอง ด้วยข้อความยืนยัน 2 ภาษา เช่น "ระบบบันทึกเวลา
   เข้างานเรียบร้อยแล้ว วันนี้ เวลา 08:30 น." และ (ข) หัวหน้าทีมของแผนกนั้น + แอดมิน HR ทุกคน ด้วย
   ข้อความสรุปว่าใครเช็คอิน/เช็คเอาท์เมื่อไหร่
2. **ยื่นคำขอลาใหม่** — แจ้งเตือนหัวหน้าทีมของแผนกพนักงานคนนั้น + แอดมิน HR ทุกคนทันทีที่มีคนยื่นคำขอลา
3. **พิจารณาคำขอลา (อนุมัติ/ไม่อนุมัติ)** — แจ้งเตือน (ก) ตัวพนักงานเจ้าของคำขอ ด้วยข้อความผลการพิจารณา
   เช่น "คำขอลาของคุณได้รับการอนุมัติเรียบร้อยแล้ว" และ (ข) หัวหน้าทีม + แอดมิน HR ทุกคน ด้วยข้อความสรุป
   ว่าใครพิจารณาผลอย่างไร
4. **(กดเอง) ส่งสรุปวันนี้เข้า LINE** — ปุ่ม "📤 ส่งสรุปวันนี้เข้า LINE" ในหน้าแอดมิน แท็บรายงาน แอดมินที่
   กดต้องเคย "เชื่อมต่อ LINE" ไว้แล้ว (ดูขั้นตอนที่ 3 ด้านล่าง) ระบบจะส่งสรุปไปที่แชทส่วนตัวของแอดมินคนนั้น
   กับ LINE OA เท่านั้น — ใช้ได้จากเบราว์เซอร์ทั่วไป ไม่จำเป็นต้องเปิดผ่านแอป LINE

ข้อความแจ้งเตือนทั้งหมดเป็น **2 ภาษา (ไทย + อังกฤษ ในข้อความเดียวกัน)** ให้ตรงกับระบบที่รองรับ 2 ภาษา
(ดูหัวข้อ "รองรับ 2 ภาษา (ไทย/อังกฤษ)" ด้านล่าง)

**สถาปัตยกรรมโดยย่อ:** เบราว์เซอร์ของพนักงาน/แอดมินเป็นคนหา `lineUserId` ของผู้ควรได้รับ (หัวหน้าทีม/
แอดมิน) จาก Firestore ตรงๆ (`js/notify.js`) แล้วเรียกไปที่ `/.netlify/functions/line-push` (ไฟล์
`netlify/functions/line-push.js`) ซึ่งเป็นจุดเดียวที่ถือ LINE Channel Access Token ไว้ (เก็บเป็น
Environment Variable ในตั้งค่าไซต์ Netlify ไม่เคยถูกส่งไปให้เบราว์เซอร์เห็น) แล้วยิงข้อความออกไปจริง —
ฟังก์ชันนี้ deploy ไปพร้อมกับตัวเว็บทุกครั้งที่คุณอัปเดตไซต์ตามปกติ **ไม่มีขั้นตอน deploy แยกต่างหาก**

**ขั้นตอนตั้งค่า (ทำครั้งเดียว):**

1. **ขอ Channel Access Token จาก LINE Developers Console** — ไปที่
   [developers.line.biz/console](https://developers.line.biz/console/) → เลือก Provider → เลือก
   **Messaging API channel** ของ LINE OA บริษัท (ช่องเดียวกับที่ผูก Rich Menu ไว้ ไม่ใช่ LIFF channel)
   → แท็บ "Messaging API" → เลื่อนลงหา "Channel access token" → กด "Issue" (หรือใช้ปุ่ม "Issue long-lived
   channel access token" ถ้ามี) → คัดลอก Token ที่ได้เก็บไว้ (ห้ามเผยแพร่ที่อื่น)
2. **ตั้งค่า Environment Variable บนไซต์ Netlify** — ไปที่ https://app.netlify.com → เลือกไซต์นี้ →
   **Project configuration** → **Environment variables** → **Add a variable** →
   - Key: `LINE_CHANNEL_ACCESS_TOKEN`
   - Value: วาง Token ที่คัดลอกจากขั้นตอนที่ 1
   - Scopes: ปล่อยค่าเริ่มต้น (ทุก scope) ก็ได้
   → กด Save แล้ว **deploy ไซต์ใหม่อีกครั้ง** (ที่แท็บ Deploys → Trigger deploy → Deploy site) เพื่อให้
   ฟังก์ชันอ่านค่าตัวแปรนี้ได้ (ตัวแปร environment จะมีผลตั้งแต่รอบ deploy ถัดไปเท่านั้น)
3. **ให้แอดมิน HR แต่ละคนกด "เชื่อมต่อ LINE" ที่หน้าแดชบอร์ดแอดมิน** — เปิด `admin.html` → เลือกชื่อ +
   กรอก PIN เข้าใช้งานตามปกติ → จะเห็นแถบสีเขียว "🔔 เชื่อมต่อ LINE ของคุณ เพื่อรับการแจ้งเตือนอัตโนมัติ"
   ด้านบนแดชบอร์ด → กด "เชื่อมต่อเลย" → ถ้ายังไม่เคยล็อกอิน LINE ในเบราว์เซอร์นี้ ระบบจะพาไปหน้าล็อกอิน
   LINE ก่อน (ล็อกอินสำเร็จแล้วจะพากลับมาที่หน้านี้ ให้เลือกชื่อ + PIN ใหม่อีกครั้ง) ทำครั้งเดียวต่อคน
   ต่ออุปกรณ์/เบราว์เซอร์ 1 ครั้งก็พอ (แถบนี้จะหายไปเองหลังเชื่อมต่อสำเร็จ) — แอดมินที่ไม่เชื่อมต่อจะไม่
   ได้รับการแจ้งเตือน แต่ยังเข้าใช้งานหน้าแอดมินได้ตามปกติทุกอย่าง
4. **ทดสอบ:** ให้พนักงานในแผนกที่มีหัวหน้าทีมกดเช็คอิน/เช็คเอาท์ 1 ครั้ง — หัวหน้าทีม (ที่ผูกบัญชี LINE
   ไว้แล้ว และเป็นเพื่อนกับ LINE OA บริษัทอยู่) และแอดมิน HR ที่เชื่อมต่อ LINE ไว้แล้ว ควรได้รับข้อความ
   แจ้งเตือนภายในไม่กี่วินาที ถ้าไม่ได้รับให้ไปดู log ที่ Netlify: หน้าไซต์ → **Logs** → **Functions** →
   เลือก `line-push` ดูสาเหตุ (เช่น ยังไม่ได้ตั้งค่า Token/Token ผิด/ยังไม่ได้ผูกบัญชี LINE/ยังไม่ได้เป็น
   เพื่อนกับ OA)

**ข้อควรรู้เพิ่มเติม:**
- พนักงาน/หัวหน้าทีมต้อง **เคยเปิดแอปผ่าน LINE อย่างน้อย 1 ครั้ง** (เพื่อให้ระบบบันทึก `lineUserId` ของ
  เขาไว้) ส่วนแอดมิน HR ต้อง **กด "เชื่อมต่อ LINE" ที่หน้าแดชบอร์ดแอดมิน** (ดูขั้นตอนที่ 3 ด้านบน) — ทุก
  คนต้อง **เป็นเพื่อนกับ LINE OA ของบริษัทอยู่ด้วย** ไม่งั้น LINE จะปฏิเสธการส่งข้อความ push ให้
- ถ้ายังไม่ได้ตั้งค่า Environment Variable ตามขั้นตอนข้างต้น ทุกอย่างอื่นในแอปยังใช้งานได้ปกติทั้งหมด
  (แค่จะไม่มีข้อความแจ้งเตือนส่งออกไปเท่านั้น การบันทึกเวลา/ยื่นคำขอ/อนุมัติ ไม่ได้รับผลกระทบใดๆ)
- แก้ไข/ปรับข้อความแจ้งเตือน หรือเงื่อนไขการส่งได้ที่ `js/notify.js` (หาผู้รับ + สร้างข้อความ) และ
  `netlify/functions/line-push.js` (ตัวส่งจริง — ไม่ค่อยต้องแก้ไฟล์นี้)
- ข้อมูลการผูกบัญชี LINE ของแอดมินเก็บอยู่ใน Firestore collection `adminLineLinks` (id เอกสาร = รหัส
  แอดมินใน `ADMINS` ของ `js/config.js`) แก้ไข/ลบได้ตรงๆ ใน Firebase Console ถ้าต้องการยกเลิกการเชื่อมต่อ
  ของแอดมินคนใดคนหนึ่ง
- ถ้าเว็บของคุณ deploy อยู่บน **Firebase Hosting** แทน Netlify (ดูขั้นตอนที่ 3 ด้านบน) ฟีเจอร์แจ้งเตือน
  LINE ในเวอร์ชันนี้จะใช้งานไม่ได้ (Netlify Functions ทำงานเฉพาะบนไซต์ Netlify เท่านั้น) — แนะนำให้ย้ายมา
  ใช้ Netlify สำหรับ hosting แทน (ฟรีเช่นกัน ดูขั้นตอนที่ 3) ทุกส่วนอื่นของแอปยังใช้งานได้ปกติไม่ว่าจะ
  deploy เว็บที่ไหนก็ตาม

  เพิ่มขึ้นมา แสดงสถานะเช็คอิน/เช็คเอาท์วันนี้ของทุกคนในแผนกแบบดูอย่างเดียว (ดูวิธีตั้งค่าที่หัวข้อ
  "แยกแผนก + หัวหน้าทีม + แจ้งเตือน LINE" ด้านบน) + ได้รับแจ้งเตือนเข้า LINE ทันทีที่ลูกทีมเช็คอิน/

  (ข้อมูลที่บันทึกจริงใน Firestore ยังคงเป็นภาษาไทยเสมอ ระบบแปลเฉพาะข้อความที่แสดงผลเท่านั้น)

**ฝั่งแอดมิน (`admin.html`)**
